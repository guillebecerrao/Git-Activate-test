# Workshop de Git — Activate Team

Um workshop prático de Git para a **Product Triad** (Product Designer, Product Engineer, Product Manager).

## Por que este workshop?

Seu time vai colaborar em um repositório compartilhado para criar especificações de produto usando **Spec-Driven Development** — uma forma estruturada de documentar requisitos e fazer o handoff para o time de engenharia. Para isso, vocês precisam entender Git o suficiente para trabalhar juntos sem perder o trabalho de ninguém.

Este workshop ensina exatamente isso: não todos os comandos do Git, mas os que importam quando se colabora em um repositório compartilhado.

> Você terá um assistente de IA (como Claude ou Codex) quando trabalhar no seu repositório real. Este workshop foi projetado para ser feito **manualmente**, para que você entenda o que está acontecendo antes de o assistente te guiar.

## Para quem é?

- Times de **3 a 5 pessoas** trabalhando juntos (recomendado)
- Ou **uma pessoa** trabalhando sozinha — cada módulo inclui um caminho solo

## Módulos

| Pasta | Tema | Nível | Tempo est. |
|---|---|---|---|
| `00-setup/` | SSH key, clonar, git config | Pré-requisito | 20 min |
| `01-primer-commit/` | status, add, commit, push | Iniciante | 20 min |
| `02-trabajo-en-equipo/` | pull, merge, resolução de conflitos | Intermediário | 30 min |
| `03-ramas/` | branch, merge, histórico | Intermediário | 30 min |
| `04-historial/` | log, diff, revert, stash | Avançado | 20 min |

## Como usar este workshop

1. **Antes da sessão:** cada participante completa `00-setup/` individualmente na sua própria máquina.
2. **Durante a sessão:** avancem pelos módulos em ordem, como grupo.
3. Leia o README de cada módulo antes de começar.
4. Siga os passos um a um — não pule à frente.
5. Se travar, há dicas no final de cada README.

## Regras

- Não há respostas erradas, apenas aprendizado.
- Você pode quebrar coisas — é para isso que o Git existe.
- Pergunte quando tiver dúvidas.
- O objetivo é entender os conceitos, não memorizar comandos.
