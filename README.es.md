# Taller de Git — Activate Team

Un taller práctico de Git para la **Product Triad** (Product Designer, Product Engineer, Product Manager).

## ¿Por qué este taller?

Tu equipo va a colaborar en un repositorio compartido para crear especificaciones de producto usando **Spec-Driven Development** — una forma estructurada de documentar requerimientos y hacer el handoff al equipo de ingeniería. Para eso necesitan entender Git lo suficiente como para trabajar juntos sin perder el trabajo de nadie.

Este taller enseña exactamente eso: no todos los comandos de Git, sino los que importan cuando se colabora en un repositorio compartido.

> Tendrán un asistente de IA (como Claude o Codex) cuando trabajen en su repositorio real. Este taller está pensado para hacerse **a mano**, para que entiendan qué está pasando antes de que el asistente los guíe.

## ¿Para quién es?

- Equipos de **3 a 5 personas** trabajando juntos (recomendado)
- O **una persona** trabajando sola — cada módulo incluye un camino para hacerlo en solitario

## Módulos

| Carpeta | Tema | Nivel | Tiempo est. |
|---|---|---|---|
| `00-setup/` | SSH key, clonar, git config | Pre-requisito | 20 min |
| `01-primer-commit/` | status, add, commit, push | Básico | 20 min |
| `02-trabajo-en-equipo/` | pull, merge, resolución de conflictos | Intermedio | 30 min |
| `03-ramas/` | branch, merge, historial | Intermedio | 30 min |
| `04-historial/` | log, diff, revert, stash | Avanzado | 20 min |

## Cómo usar este taller

1. **Antes de la sesión:** cada participante completa `00-setup/` individualmente en su propia máquina.
2. **Durante la sesión:** avancen por los módulos en orden, como grupo.
3. Lee el README de cada módulo antes de empezar.
4. Sigue los pasos uno a uno — no te adelantes.
5. Si te trabas, hay pistas al final de cada README.

## Reglas

- No hay respuestas incorrectas, solo aprendizaje.
- Puedes romper cosas — para eso existe Git.
- Pregunta cuando tengas dudas.
- El objetivo es entender los conceptos, no memorizar comandos.
